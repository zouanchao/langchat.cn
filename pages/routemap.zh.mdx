## Web SDK & 消息渠道商

目前，我们预开发了一套 [LangChat Web SDK](http://github.com/tycoding/langchat-web-sdk) 项目，用于作为第三方Web端LLM Bot的快速集成，例如：客服机器人、企业私有知识库等。

[LangChat Web SDK](http://github.com/tycoding/langchat-web-sdk) 将使用原生JavaScript开发，并借助Vite将整个项目打包为一个JS文件（文件中包含了js/css/html代码）。

打包后的JS文件，可以在任意的第三方Web系统中引入，然后在页面右下角展示一个聊天机器人入口。当初始化机器人信息后，第三方应用便可以零侵入性地集成聊天机器人。

预览效果：

对于第三方网页，只需要嵌入JS即可：

```html copy
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>LangChat WebSDK Example</title>
    <script src="../dist/langchat_web_sdk.min.js"></script>
</head>
<body>
    <div>This is custom page container...</div>
    <script>
        // Configuring sdk Information
        new initLangChat('key', ...)
    </script>
</body>
</html>
```

## Workflows

其实一开始我的想法是基于WorkFlow机器人，而不是现在这样，通过不同节点（模型配置、Prompt配置、知识库配置）进行繁琐的配置。

之前考虑过使用Vue-Flow和LiteFlow开发，但当时一直没有好的流程编排执行方案，导致项目拖延至今。
现在，我考虑在现有功能基础上单独开发 **langchat-flow** 模块，提供一种快速、高度自定义的编排方式来定制和发布机器人应用。
